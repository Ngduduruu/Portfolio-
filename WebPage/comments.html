HTML Structure

The HTML structure is the main framework of the website using the standard markup language, organising content into semantic elements like headers, navigation, body, sections, and footers. The webpage is declared as a standardised HTML5 document using the declaration at the beginning of the file. A more detailed breakdown of the structure is as follows:

a. header

The header section is defined using the <header> tag, which contains a heading element <h1> that displays the title of the website. The title is displayes as "Web System Assignment". The website title is written using the <title> tag in the <header> section. Below the title, there is a navigation bar that allows users to navigate to different pages of the website. The navibation bar is created using the <nav> tag and contains links that are labelled as "Home", "Past", "Future" and "Comments". Users can click on these links to navigate to the respective pages. The section is designed to be consistent across all pages of the website, providing a seamless user experience.

b. nav

The navigation section is defined using the <nav> tag, which contains a list of links to the different pages of the website. This section is typically used to provide easy access to the main sections of the website. Users can access the Home, Past, Futute and Comments tab page through the navigation bar by clicking on the respective links. The <a> tag is used to create hyperlinks to the different pages, and the href attribute specifies the URL of the page that the link goes to.

c. main

The main content section is defined using the <main> tag, which contains the unique content for each page. This section is where the main information and content of the webpage is displayed. Each page has its own unique content that is relevant to the title of the page. For example, the Home page contains an introduction and some fun facts about me, while the Past and Future pages contain information about my past experience and future goals in computing, respectively. The content is structured using various HTML tags, such as <h2>, <p>, <ul>, <li>, <table> and <article>. Using the <h2> tag to create subheadings within the main content and the <p> tag to create paragraphs of text. The content is organised in a way that is easy to read and navigate, providing a clear structure for users to follow. The <article> tag is used to encapsulate the subheadings and paragraphs within the <div> element.

d. footer

The footer section is defined using the <footer> tag, which contains copyright information. This section is typically used to provide additional information or links at the bottom of the webpage. In this case, it displays the copyright information for the website.

Other HTML Attributes

In addition to the main structural tags, there are also other HTML attributes used in the code.

Attribute Description

CSS Structure 🎨

The design and styling of the website is defined using the CSS file, which is linked to the HTML files using the <link> tag in the <head> section. The CSS file contains various styles and rules that define the visual appearance of the website. Each section contains specific styles that are applied to the respective elements in the HTML files. For example, the header section contains styles for the <header> tag, while the navigation section contains styles for the <nav> tag. The CSS file also includes styles for specific classes and IDs that are used in the HTML files.

Font Style and Size

The font style used in the website is defined using the font-family property in CSS. The font used is "Arial, sans-serif", which is a common and widely supported font that provides a clean and modern look to the website. The font size is defined using the font-size property, ensuring that the text is appropriately sized for readability across different devices and browsers. This particular font choice and size contribute to the overall aesthetic and readability of the website while maintaning a modern and elegant look.

Color Scheme

The color scheme of the website is defined using various CSS properties such as background-color, color, and border-color. The website features a combination of neutral colour palettes such as white, gray, and beige, with accents of brown and black. The colour scheme represents a sense of elegance that highlights the whites by evoking cleanliness, sophistication and simplicity. The use of neutral colors also allows the content to stand out and be easily readable, while the accents of brown and black add a touch of warmth and contrast to the overall design. The use of contrasting colors for text and background ensures that the content is easily readable and visually appealing. To achieve a perfect contrast ratio of a minimum of 6:1, this contrast checker is used to ensure that the color choices meet accessibility standards.

Layout and Design

The layout and design of the website is defined using various CSS properties such as display, flex, margin, and padding. The website uses a responsive design that adapts to different screen sizes and devices. The use of flexbox allows for a flexible and dynamic layout that can adjust to the content and screen size. The use of margins and padding helps to create a visually appealing and well-structured layout that enhances the user experience. The table elements are styled to follow the core design principles of minimalism, with alternating row colors for better readability.

Accessibility Considerations

Accessibility is an important aspect of web design, some several considerations have been taken into account to ensure that the website is accessible to all users. This includes using semantic HTML tags to provide structure and meaning to the content, providing alternative text for images using the alt attribute, and ensuring that the color contrast is sufficient for users with visual impairments. Additionally, I have used responsive design techniques to ensure that the website is usable on different devices and screen sizes, which can benefit users with various accessibility needs.

Transition

To enhance the user experience, I have implemented smooth transition effects for the links on the navigation bar. This is achieved using the transition property in CSS, which allows for a gradual change in styles when a user interacts with an element. For example, when a user hovers over a link, the color may change into a slight shade of grey. If user is on the current page, the link will be highlighted with a different color to indicate the active state, providing a clear visual cue for navigation. Besides that, a simple fade-in transtion is applied to the page when it loads.

Validation of Work

To ensure the validity of the HTML and CSS code, I have used the W3C Markup Validation Service and the W3C CSS Validation Service. These tools help to identify any errors or issues in the code and provide suggestions for improvement. By using these validation services, I can ensure that the website is built using best practices and adheres to web standards, which can improve the overall performance and compatibility of the website across different browsers and devices.
